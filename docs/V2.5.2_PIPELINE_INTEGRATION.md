# V2.5.2 — Pipeline Integration

This stage connects the previously merged components into one auditable research cycle:

1. Load or create a Living Investment File.
2. Collect evidence from the existing daily snapshot.
3. Persist and deduplicate evidence.
4. Score every evidence record.
5. Save score details into evidence metadata.
6. Regenerate scenarios only when evidence changed.
7. Save the updated Living Investment File once per changed cycle.
8. Write an execution log for every run.

## Safety rules

- Missing financial values remain `None`.
- Evidence score measures evidence strength, not investment attractiveness.
- Evidence direction remains separate from evidence strength.
- Scenario regeneration is skipped when no evidence was created, updated, or newly linked.
- A collector or scoring failure is logged instead of silently terminating the whole cycle.

## Commands

Run the existing daily discovery pipeline first:

```bash
python scripts/run_daily_pipeline.py
```

Then run the research cycle:

```bash
python scripts/run_research_pipeline.py data/todays_opportunities.json
```

## Outputs

- Living files: `data/investment_files/`
- Evidence: `data/evidence/`
- Research run logs: `data/research_runs/`

External web research and Brave Search remain outside this stage.
