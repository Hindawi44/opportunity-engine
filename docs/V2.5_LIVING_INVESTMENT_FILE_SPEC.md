# V2.5.0 — Living Investment File Specification

## الهدف

تحويل كل فرصة مكتشفة إلى ملف استثماري حي قابل للتحديث، يفصل بوضوح بين الحقائق والافتراضات والأدلة والمعلومات الناقصة، ويعرض عدة مسارات للاستفادة دون إصدار قرار شراء آلي.

## المبدأ الحاكم

الدراسة الاستثمارية هي المنتج النهائي. درجات التقييم وWATCH وREJECT تبقى إشارات داخلية مساعدة فقط.

## دورة حياة الملف

1. `discovered` — تم اكتشاف الفرصة.
2. `normalized` — تم توحيد بياناتها وإزالة التكرار.
3. `researching` — يجري جمع الأدلة والسوق والمشترين.
4. `scenario_ready` — يوجد مسار استفادة واحد على الأقل قابل للفحص.
5. `needs_evidence` — التحليل متوقف على معلومات ناقصة جوهرية.
6. `monitoring` — الملف ينتظر تغير السعر أو ظهور دليل جديد.
7. `archived` — انتهت الفرصة أو لم تعد قابلة للمتابعة.

## الأقسام الإلزامية

### 1. الهوية والمصدر

- `opportunity_id`
- `schema_version`
- `status`
- `title`
- `source_name`
- `source_url`
- `source_item_id`
- `first_seen_at`
- `last_seen_at`
- `last_analyzed_at`

### 2. الأصل المعروض

- الفئة
- الوصف
- الموقع
- السعر والعملة
- الكمية
- الحالة
- البائع
- الصور
- الموعد النهائي أو نهاية المزاد

### 3. الحقائق المؤكدة

كل حقيقة يجب أن تحتوي على:

- نص الحقيقة
- مصدرها
- وقت جمعها
- درجة الثقة
- هل تم التحقق منها من مصدر ثانٍ؟

### 4. الافتراضات

كل افتراض يجب أن يحتوي على:

- نص الافتراض
- سبب ظهوره
- أثره المحتمل
- طريقة التحقق
- الحالة: مفتوح، مؤكد، مرفوض

### 5. الأدلة

أنواع الأدلة الأولية:

- سعر سوق
- طلب محتمل
- مشتري محتمل
- تكلفة نقل
- تكلفة تخزين
- تكلفة فك أو تجهيز
- قيد قانوني أو ضريبي
- خبرة سابقة مشابهة

### 6. المعلومات الناقصة

لكل معلومة ناقصة:

- السؤال
- سبب أهميته
- الجهة أو المصدر المحتمل للإجابة
- الأولوية
- هل تمنع التقدم؟

### 7. العملاء أو المشترون المحتملون

- القطاع
- نوع المشتري
- المنطقة
- سبب الملاءمة
- دليل الاهتمام
- وسيلة الوصول
- درجة الثقة

### 8. مسارات الاستفادة

المسارات القياسية:

- شراء وإعادة بيع
- وساطة دون تملك
- شراكة تمويل أو تخزين
- تقسيم الصفقة
- بيع مسبق
- إدارة التصفية مقابل نسبة
- تأجير أو إعادة استخدام تشغيلي

كل مسار يحتوي على:

- الفكرة
- العميل النهائي
- طريقة تحقيق الإيراد
- رأس المال المطلوب
- التكاليف المتوقعة
- الإيراد المحافظ
- الربح المحتمل
- المدة
- المخاطر
- وسائل التخفيف
- الأدلة المؤيدة
- الأدلة المعارضة
- شروط القابلية للتنفيذ
- أول خطوة عملية

### 9. الاختبار الصغير

يجب أن يقترح النظام تجربة منخفضة التكلفة قبل الالتزام الكبير، مثل:

- الاتصال بثلاثة مشترين محتملين
- عرض جزء من المخزون مسبقًا
- طلب قائمة جرد وصور إضافية
- التفاوض على دفعة أصغر
- الحصول على عرض نقل مكتوب

### 10. التوصية الحالية

لا تحتوي على BUY أو REJECT نهائي. يجب أن تحتوي على:

- أفضل مسار حالي
- سبب اختياره
- مستوى الثقة
- ما الذي قد يغيره
- الخطوة التالية

### 11. سجل التحديث

كل تحديث يسجل:

- وقت التحديث
- نوع التغيير
- الحقول المتغيرة
- السبب
- المصدر
- هل أعيد التحليل؟

## قواعد النزاهة

1. لا تتحول القيمة المفقودة إلى صفر.
2. لا يكتب الافتراض بصيغة حقيقة.
3. لا يستخدم دليل دون رابط أو مرجع مصدر.
4. لا يحذف التاريخ السابق عند التحديث.
5. لا يصدر شراء أو مزايدة أو تواصل آلي.
6. لا يعاد البحث عبر Brave عندما تكون نتيجة صالحة موجودة في الذاكرة المؤقتة.
7. كل مبلغ مالي يجب أن يحمل العملة ونوعه: معروض، تقديري، مؤكد، أو محافظ.

## معايير اكتمال V2.5.0

- يوجد JSON Schema رسمي للملف.
- يمكن إنشاء ملف جديد من فرصة موحدة.
- يمكن تحديث الملف دون فقد التاريخ.
- يتم الفصل بين الحقائق والافتراضات والمفقودات.
- يدعم عدة مسارات استفادة.
- يدعم خطة اختبار صغيرة وخطوة تالية.
- توجد اختبارات تحقق من الحقول الإلزامية وسلامة التحديث.
- لا توجد أي عملية شراء أو مزايدة تلقائية.
