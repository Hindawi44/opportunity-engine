# V2.5.2 — Evidence Model & Store

## الهدف

إنشاء طبقة موحدة لحفظ الأدلة البحثية لكل فرصة قبل بناء موصلات البحث الخارجية.

## أنواع الأدلة

- `market_price`: أسعار ومقارنات السوق.
- `buyer`: مشترون وشركات محتملة.
- `cost`: تكاليف صريحة.
- `demand`: مؤشرات الطلب.
- `legal`: متطلبات وقيود قانونية.
- `seller`: معلومات موثقة عن البائع.
- `logistics`: نقل، تخزين، فك وتحميل.
- `other`: دليل لا يطابق الفئات السابقة.

## قواعد الثقة والاتجاه

كل دليل يحمل مستوى ثقة `low / medium / high`، واتجاهًا يوضح هل يدعم السيناريو أو يضعفه أو يبقى محايدًا.

## منع التكرار

يُنشأ fingerprint ثابت من:

- رقم الفرصة.
- نوع الدليل.
- نص الدليل بعد التطبيع.
- اسم المصدر.
- رابط المصدر.

عند ظهور الدليل نفسه مرة أخرى، لا يُنشأ سجل جديد؛ بل تضاف Observation جديدة ويحفظ تاريخ القيم والتغيرات.

## حفظ البيانات

المسار الافتراضي:

```text
data/evidence/<opportunity_id>/<evidence_id>.json
```

الحفظ ذري باستخدام ملف مؤقت ثم استبدال الملف النهائي.

## قواعد النزاهة

- القيم المالية المفقودة تبقى `None` ولا تتحول إلى صفر.
- القيم الرقمية السالبة مرفوضة.
- روابط المصادر يجب أن تستخدم HTTPS.
- معرفات الفرص التي قد تسمح بالخروج من مجلد البيانات مرفوضة.
- كل تحديث يحتفظ بالملاحظات السابقة بدل الكتابة فوقها.

## حدود هذه الدفعة

هذه الدفعة تنفذ المرحلتين 2.1.1 و2.1.2 فقط:

- Evidence Data Model.
- Evidence Store.

لا تتضمن بعد:

- البحث عبر Brave أو أي مصدر خارجي.
- تقييم قوة المصدر آليًا.
- الربط التلقائي بالـPipeline.
- إعادة تشغيل Scenario Generator.
