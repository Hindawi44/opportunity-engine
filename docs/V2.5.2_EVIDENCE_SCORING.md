# V2.5.2 — Evidence Scoring Engine

## الهدف

قياس قوة الدليل من 0 إلى 100 دون تحويله إلى قرار شراء أو ضمان نجاح. الدرجة تقيس **جودة الدليل** فقط، وليس جاذبية الفرصة الاستثمارية.

## الأبعاد والأوزان

| البعد | الحد الأقصى |
|---|---:|
| جودة المصدر | 25 |
| حداثة الدليل | 15 |
| اكتمال السجل | 15 |
| قابلية التحقق الرقمي | 15 |
| التوافق مع أدلة مستقلة | 15 |
| أولية المصدر | 10 |
| الصلة بالسيناريو | 5 |
| **المجموع** | **100** |

## التصنيف

- 0–39: ضعيف
- 40–69: متوسط
- 70–89: قوي
- 90–100: موثوق جدًا

## طبقات المصادر

1. `primary_official`: مصدر حكومي أو سجل رسمي.
2. `primary_commercial`: الإعلان الأصلي أو منصة المزاد أو السوق الأساسي.
3. `secondary_reputable`: مصدر ثانوي معروف وموثوق نسبيًا.
4. `secondary_unverified`: موقع أو مصدر ثانوي غير متحقق.
5. `unknown`: لا توجد معلومات كافية عن المصدر.

يمكن تحديد طبقة المصدر صراحة داخل `metadata.source_tier`. وإذا لم توجد، يحاول المحرك تصنيفها من النطاق.

## قواعد مهمة

- الدليل الذي يضعف سيناريو لا يحصل تلقائيًا على درجة أقل؛ الاتجاه شيء، وقوة الدليل شيء آخر.
- القيمة الرقمية المفقودة تبقى `None` ولا تعامل كصفر.
- دليل السعر أو التكلفة دون رقم موثق لا يحصل على نقاط التحقق الرقمي.
- التوافق يحتسب فقط من مصادر مستقلة أخرى لنفس الفرصة ونوع الدليل.
- الدرجة حتمية وقابلة للمراجعة عبر `breakdown`, `reasons`, و`warnings`.
- هذه الدفعة لا تغيّر ملفات الأدلة ولا تربط التقييم تلقائيًا بالـPipeline؛ ذلك ضمن 2.1.5.

## معايير الاكتمال

- إرجاع درجة بين 0 و100.
- إرجاع التصنيف المناسب عند الحدود الأربعة.
- توضيح تفاصيل الدرجة لكل بعد.
- التمييز بين المصدر الأولي والثانوي.
- احتساب الحداثة والتوافق والاكتمال.
- عدم اختراع قيمة مالية أو دليل جديد.
- وجود اختبارات للحدود، القدم، المصادر، القيم المفقودة والتوافق.
