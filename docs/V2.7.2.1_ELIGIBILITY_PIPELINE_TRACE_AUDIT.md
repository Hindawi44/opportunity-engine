# V2.7.2.1 — Eligibility & Pipeline Trace Audit

## الهدف
تحديد سبب توقف كل فرصة حقيقية قبل البحث الخارجي أو الأدلة أو السيناريوهات، دون إنشاء محرك استثماري جديد ودون تغيير قواعد الشراء.

## المخرجات الجديدة لكل فرصة

- `external_research_eligible`
- `eligibility_reason`
- `internal_score`
- `required_score`
- `pipeline_stage_reached`
- `blocked_before`

## أسباب التأهل والاستبعاد

- `explicitly_eligible`
- `explicitly_ineligible`
- `missing_internal_score`
- `internal_score_below_threshold`
- `internal_score_meets_threshold`

## مراحل التتبع

1. `eligibility`
2. `external_research`
3. `evidence`
4. `scenario_generation`
5. `best_scenario`

## مؤشرات التدقيق

يضيف التقرير أعداد الفرص:

- المستبعدة بسبب انخفاض الدرجة.
- المستبعدة بسبب غياب الدرجة.
- المستبعدة صراحة.
- التي وصلت إلى البحث الخارجي.
- التي وصلت إلى الأدلة.
- التي وصلت إلى السيناريوهات.

## قاعدة النجاح

نجاح GitHub Actions يعني نجاحًا تقنيًا فقط. النجاح التحليلي يتطلب أن يوضح التقرير أين توقفت كل فرصة، وأن تصل فرصة مؤهلة واحدة على الأقل إلى مرحلة خارجية في تشغيل لاحق.

## خارج النطاق

- تخفيض حد التأهل تلقائيًا.
- إجبار فرصة ضعيفة على استخدام Brave.
- اختراع دليل أو مشترٍ أو سعر.
- إصدار قرار شراء أو مزايدة أو تواصل آلي.
