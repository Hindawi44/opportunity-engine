# V2.6.1 — Brave Search Connector

## Objective

Provide a bounded, auditable connector to Brave Web Search for later external-research stages.

This release performs search and normalization only. It does **not** calculate market value, discover buyers, update Living Investment Files, or select investment scenarios.

## Official API contract used

- Endpoint: `GET https://api.search.brave.com/res/v1/web/search`
- Authentication header: `X-Subscription-Token`
- Required query: `q`
- Query limit: maximum 400 characters and 50 words
- Result count: 1–20
- Optional targeting: country, search language and freshness

## Cost and usage controls

- Default maximum: 8 real API requests per process run.
- Default cache lifetime: 24 hours.
- Repeated identical searches use cache and do not consume a second API request.
- Usage is written to `data/brave_usage.jsonl`.
- The usage log stores a query hash, never the plain query or API key.
- Tests use injected fake transports and never call the live Brave API.

## Environment variables

```text
BRAVE_API_KEY=...
BRAVE_MAX_REQUESTS_PER_RUN=8
BRAVE_CACHE_TTL_HOURS=24
BRAVE_CACHE_DIR=data/brave_cache
BRAVE_USAGE_LOG=data/brave_usage.jsonl
BRAVE_TIMEOUT_SECONDS=20
```

`BRAVE_SEARCH_API_KEY` is also accepted for compatibility.

## Normalized result fields

Each usable result contains:

- stable result ID (Brave ID or URL-derived fallback)
- title
- HTTPS URL
- snippet
- extra snippets
- source label
- published/page age when supplied
- language when supplied
- source rank

Duplicate URLs, missing titles and non-HTTPS results are discarded.

## Error handling

- Authentication failures become clear runtime errors.
- HTTP 429 becomes a rate-limit error.
- Network and malformed-JSON errors do not expose the API key.
- Reaching the local per-run budget stops further real requests before calling Brave.

## Completion criteria

- Authorized request construction is tested.
- Results are normalized without invented values.
- Duplicate and unsafe results are rejected.
- Cache reuse is tested.
- Per-run request budget is tested.
- Usage logging is tested for secret/query privacy.
- GitHub Actions passes before merge.
