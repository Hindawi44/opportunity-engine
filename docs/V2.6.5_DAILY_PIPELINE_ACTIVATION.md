# V2.6.5 — Daily Pipeline Activation

This stage activates bounded external research after discovery and Living Investment File synchronization.

## Execution order

1. Daily discovery snapshot.
2. Living Investment File synchronization.
3. Rank eligible opportunities by the existing internal score.
4. Select at most `DAILY_EXTERNAL_MAX_OPPORTUNITIES` opportunities (default target: 3).
5. Run the External Evidence Loop for each selected file.
6. Save each updated file independently.
7. Report searches, skipped searches, cache hits, failures, and errors.

## Safety rules

- External research is disabled when the Brave API key is unavailable.
- One opportunity failure never aborts the remaining daily run.
- Selection is bounded and deterministic.
- The External Evidence Loop remains responsible for missing-information detection and avoiding unnecessary searches.
- Brave remains responsible for per-run request limits and cache reuse.
- No buyer contact, bidding, purchase, or final automatic investment decision is permitted.

## Output fields

The daily response will expose activation status, selected/completed/failed opportunities, searches executed/skipped, cache hits, and non-fatal errors.
