# V2.7.1 — Real Dataset Validation

## الهدف

قياس مخرجات النظام على بيانات فرص حقيقية بدل الاكتفاء بنجاح التشغيل التقني.
هذه المرحلة لا تغيّر التوصيات، ولا تشتري، ولا تزايد، ولا تتواصل مع أي مشترٍ.

## المدخل

الملف الافتراضي:

```text
data/todays_opportunities.json
```

يمكن أن يحتوي على قائمة مباشرة، أو مفاتيح مثل `opportunities`, `rows`, `items`, `results` أو `data`.

## التشغيل

```bash
PYTHONPATH=src python scripts/run_real_opportunity_validation.py \
  data/todays_opportunities.json \
  --output data/validation/v2.7.1-real-dataset-report.json
```

## مؤشرات الأداء المسجلة

- عدد الفرص المكتشفة والفريدة.
- عدد التكرارات.
- تغطية رابط المصدر والسعر.
- عدد الفرص المؤهلة للبحث الخارجي.
- عدد المقارنات المقبولة وتغطيتها.
- عدد المشترين المحتملين وتغطيتهم.
- تغطية الأدلة والسيناريوهات.
- وجود أفضل سيناريو.
- متوسط الدرجة الداخلية.
- مدة إنشاء التقرير.

## حدود هذه المرحلة

هذه الوحدة تقيس ما هو موجود فعليًا في المخرجات فقط. لا تستنتج سعرًا مفقودًا، ولا تعتبر المشتري المحتمل مشتريًا مؤكدًا، ولا تحكم بعد على دقة السعر أو جودة المشتري. هذه المهام مخصصة للمراحل V2.7.2 وV2.7.3.

## معيار الاكتمال

- ينجح التقرير على بيانات حقيقية دون تعديل يدوي للملف.
- تبقى القيم المفقودة مفقودة ولا تتحول إلى صفر.
- تظهر التكرارات ومشكلات التغطية بوضوح.
- تكون النتيجة JSON قابلة للمقارنة بين تشغيل وآخر.
