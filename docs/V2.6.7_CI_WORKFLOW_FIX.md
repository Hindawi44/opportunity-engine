# V2.6.7 — CI Workflow Fix

## Problem
The guarded live dry run failed during the installation step because the repository is not packaged with `setup.py` or `pyproject.toml`, while the workflow used `pip install -e .`.

## Fix
- Install dependencies from `requirements.txt`.
- Set `PYTHONPATH` to the repository `src` directory so project modules can be imported without editable installation.
- Keep the workflow manual and preserve all existing Brave limits and artifact uploads.

## Safety
This change does not add automated purchasing, bidding, or buyer contact. It only corrects CI runtime setup for the existing dry run.

## Validation
After merge, manually run **V2.6.6 Live Dry Run** on `main` and inspect the uploaded artifact before declaring production readiness.
