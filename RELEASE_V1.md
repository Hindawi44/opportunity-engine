# Opportunity Engine v1.0 — Production Runbook

## التشغيل الآمن

1. استخدم Python 3.11 أو أحدث وثبّت `requirements.txt`.
2. انسخ `.env.example` إلى ملف بيئة محلي، ولا ترفع الأسرار إلى GitHub.
3. فعّل فقط الموصلات التي لديك صلاحية رسمية لاستخدامها.
4. شغّل الاختبارات: `pytest -q`.
5. شغّل فحص الصحة: `python scripts/production_health_check.py`.
6. شغّل خط الفرص: `python scripts/run_daily_pipeline.py`.
7. راجع `data/todays_opportunities.json` و`data/smart_alerts.json` يدويًا قبل أي قرار مالي.
8. شغّل تخصيص رأس المال فقط بعد تحديد رأس المال الحقيقي.
9. سجّل أي شراء أو بيع يدويًا عبر `scripts/manage_portfolio.py`.

## معايير الجاهزية

- جميع الاختبارات ناجحة.
- تقرير الصحة يحمل `ready: true`.
- لا توجد مجموعة أسرار مهيأة جزئيًا.
- مجلد البيانات قابل للكتابة.
- لا توجد بيانات تجريبية مستخدمة كحقائق سوق.
- المقارنات والتكاليف المفقودة تبقى مفقودة ولا تتحول إلى صفر.

## حدود الإصدار

- لا يوجد شراء أو مزايدة أو بيع تلقائي.
- التنبيهات تُنشأ في Outbox؛ قناة البريد أو Telegram تحتاج إعدادًا منفصلًا.
- الموصلات الاختيارية لا تعمل دون بيانات اعتماد مصرح بها.
- التوصيات أدوات دعم قرار وليست ضمانًا للربح.

## ملفات التشغيل الرئيسية

- `data/todays_opportunities.json`: لقطة الفرص اليومية.
- `data/price_history.json`: سجل تغير الأسعار.
- `data/smart_alerts.json`: سجل التنبيهات ومنع التكرار.
- `data/capital_allocation.json`: خطة رأس المال المحافظة.
- `data/portfolio.json`: سجل المحفظة والمعاملات اليدوية.
- `data/health_report.json`: تقرير سلامة التشغيل.
